1-2 HTML - Hypertext Markup Language (the heart of the web)

HTML is made up of text content, which is what you see. It is made up of markup, which is what that content looks like or how it is arranged. It is made up of references to other documents, like images and videos, or any other files. And it can be made up of links to other pages.

Plain text is plain text in HTML, for the most part. But if we want to make the text look different, we need to use something called markup, which is one of the other facets of HTML.

1-3 Tags and Elements

HTML markup is made up of things called tags. The way a tag is structured: you have got, a less than symbol, the tag name, a greater than symbol, the tag contents, which can actually be other tags or it could just be text, another less than symbol, a slash, the name, and a greater than symbol.

The first one is the opening tag, the last one, with the slash, is the closing tag. The whole thing is an element.

Bold tag everything that is between these tags appears bold.

emphasis tag everything that is between these tags appears italic.

Html can be nested.

3 Strucured Documents and CSS Style

3-1 Strucured Documents

Computer languages demand a logic made explicit by strict syntax constructions with grammar signs (semantics).

The code structure exists in computer languages as a necessary organization tool. The strict denotative learning capability of the computers demands divisors and markers; they enable the computer to understand the transitions and the syntax of the code.

A structured HTML document is a document that uses inline and block elements, classes and ids to create an structure capable of show to browsers differentiations and padronization to be adopted at each element of the page.

It's best to use semantic elements, so the browser will be able to know the function of each element at the page. In that way, we enable the browser to identify the function of each element of the page, generating a behavior coherent with the HTML code. More than that, a structured document with semantic elements makes viable all sort of automatic handling of the content, including the full power use of Artificial Intelligence.

3-2 CSS - Cascading Style Sheets

CSS, Cascading Style Sheets, is a separate language for adding styles to our documents, establishing how the page will be displayed. HTML and CSS separation exists to simplify the web pages. With that separation, we can change the style of web pages without change the HTML code.

We can have several CSS files or sheets that will be referenced in HTML and all of them will be used to find out the final and visible style for our page.

CSS uses the elements, classes and ids to correlate with HTML code. It will apply the more specific style, since the cascading feature of CSS is guided by the specificity principle. In other words: the most specific rule is applied to every element.

To supply rules where there is none, CSS uses the inheritance feature. This feature will apply some ancestor element rules to its descendants at cases established a priori.

Wikipidea's article says that: In general, descendant elements inherit text-related properties, but box-related properties are not inherited. Properties that can be inherited are color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing. Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index.

3-3 Avoiding Repetition

All softwares occupy space. Avoiding repetition we save space.

All softwares need to be processed. Avoiding repetition we simplify the program, using less processing power.

A structured code without unnecessary repetitions and with the correspondent indentation is more readable by humans, it's clearer and easier to understand, evolve and debug.

The semantic structure of HTML and the correct use of ids and classes (repeating the semantic function or class of elements of the same nature or for which we want the same formatting) enables the CSS code to avoid unnecessary repetitions.

The fact that certain properties are inherited from other elements avoids the need for a specific formatting code, if inherited format is already optimal.
